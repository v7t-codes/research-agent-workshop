Why this matters to you

The practitioner case

Scale your output
  • Research 10x faster — literature reviews in minutes, not days
  • Run parallel threads — multiple agents exploring simultaneously
  • Auto-verification — a second process audits every claim
Apply to real problems
  • Domain-agnostic — company building, diligence, market analysis
  • Any knowledge-intensive task becomes agentic
  • The harness is reusable — swap the domain, keep the system
The compounding edge
  • Cursor, Sierra, Decagon do this at scale — you're building a narrow version
  • The window is still open — most practitioners haven't built this yet
  • Capability is cheap; closing the gap to useful output is the skill
The harness gets absorbed into the platform over time. The durable skill is knowing how to close the gap between what a model can do and what produces useful output.

First half: we show it

Live demo — 6 steps
0 4:00 Raw ClaudeNo setup. Baseline score.
1 4:05 CLAUDE.md20-line context file. Domain, sources, constraints.
2 4:12 SKILL.mdResearch methodology. Decompose, search, synthesize.
3 4:20 MCP server50 lines of Python. Live arXiv and Semantic Scholar.
4 4:30 Iterative skillLoop logic. Claude evaluates its own coverage.
5 4:40 Verification agentSeparate Claude subprocess. Audits every citation.
6 4:50 Team of agentsSearcher → Critic → Synthesizer. Files, not shared memory.
What you'll see
  • Live API calls to arXiv — real 2025 papers fetched in session
  • A second process spawning — the verification agent runs in its own context
  • Agents writing files — searcher, critic, synthesizer communicate through disk
The key shift
  • Steps 1–4 run in one session — same context window, same agent
  • Step 5 spawns an independent subprocess — a fresh Claude with no history
  • That independence is why verification works — no confirmation bias from the original session
03 / 04
5:00 – 7:00 PM

Second half: you build it

Groups of 2–3 · 90 minutes
What you're building
  • A research agent for your own domain — not ours
  • Your own CLAUDE.md, SKILL.md, and a custom MCP tool calling a real API
  • Agent architecture: verification subprocess or full searcher → critic → synthesizer team
One rule
  • You can read the demo as reference — you cannot copy it
  • Different domain, different methodology, different tool — write it yourself
The competition
  • At 6:30 PM, your agent runs against 3 unseen research questions
  • Average RACE score across all 3 = your result
  • Top 3 present: score progression, one design decision, one surprise
  • Audience votes separately for best design — a creative tool that scores 70 beats a copied one that scores 80
What wins
  • Highest score — comprehensiveness, insight, verified citations
  • Best design — the most interesting architectural choice, not the highest number
